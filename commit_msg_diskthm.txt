Thread theorem name through disk_thm for proof traces

Change disk_thm to take the theorem name as a parameter, so ORACLE
DISK_THM entries in proof traces record (theory, name) instead of
the full theorem statement. This avoids interning ancestor theorem
terms in per-theory traces, and enables the future merge tool to
resolve ancestor references by name.

Signature change:
  disk_thm : (depdisk * string list) * term list -> thm
  =>
  disk_thm : string -> (depdisk * string list) * term list -> thm

SharingTables already has the name at the call site. ThyDataSexp
(rare path, only used by EmitML) looks up the name from the DB
using the depid in the theorem's tag.

The ancestor context in ReplayTrace is changed from conclusion-based
matching to (theory, name)-based lookup, matching the new trace format.
