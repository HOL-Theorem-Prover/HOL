# Numbered-chapter ordering for the Logic manual.
#
# Each top-level (un-indented) line is a chapter stem; lines with
# leading whitespace are nested sub-files of the most recent
# top-level stem.  Front-matter chapters (title, preface) and the
# tableofcontents marker are still hand-coded in logic.tex because
# their ordering relative to \tableofcontents matters.
# Lines starting with `#` and blank lines are ignored.
#
# .smd source conventions and build behaviour: see
# Manual/Description/chapters.txt for the canonical write-up.
#
# The `references` stem at the bottom is mdbook-only: the
# bibliography chapter is generated by `smdpp render-bib` from
# logic.bib (no .smd source).  `gen_chap_lists latex-include` skips
# it because the PDF gets its bibliography from natbib's
# `\bibliography{logic}` directly.
